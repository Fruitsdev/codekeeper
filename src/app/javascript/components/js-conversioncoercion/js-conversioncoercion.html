Conversion & Coercion

Learn the difference between manual type conversion and automatic coercion in JavaScript.

Type Conversion (Manual)

  • Conversion means we convert a value from one type to another manually.
  • Common conversion functions: Number(), String(), Boolean() 
// String to Number
console.log(Number("10"));      // 10
console.log(Number("Tom"));// NaN

// Number to String
console.log(String(25));        // "25"

// Any value to Boolean
console.log(Boolean(0));        // false
console.log(Boolean(1));        // true
console.log(Boolean(""));       // false
console.log(Boolean("Hi"));     // true

Type Coercion (Automatic)

  • Coercion means JavaScript automatically converts types behind the scenes.
  • Usually happens with operators like +, -
// + converts to string if one operand is string
console.log("10" + 5);   // "105"

// - converts to number if possible
console.log("10" - 5);   // 5

console.log("10" * 2);   // 20
console.log("10" / 2);   // 5

Common Coercion Examples

console.log("5" + 1);     // "51"
console.log("5" - 1);     // 4
console.log("5" * "2");   // 10

console.log(true + 1);    // 2  (true becomes 1)
console.log(false + 1);   // 1  (false becomes 0)

NaN (Not a Number)

NaN is a number type, but it represents an invalid numeric result. 

console.log(Number("abc")); // NaN
console.log(typeof NaN);    // "number"
